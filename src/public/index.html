AgentProxy v0.0.0

AgentProxy 是一个 API 代理监控工具，支持反向代理和正向代理两种模式，可拦截并查看所有经过的 HTTP/HTTPS 请求与响应。

反向代理（默认）

客户端直接请求代理服务器，代理转发到目标地址。适用于 API 调试场景。

agentproxy -t http://10.30.6.49:9090

代理默认转发到 https://api.deepseek.com，可通过 -t 指定目标地址。在代码中将 API 地址改为 http://localhost:3000，所有请求会被转发到目标服务器。

正向代理

在系统或浏览器中设置代理服务器，浏览器所有流量自动经过代理。适用于 Web 调试场景。

agentproxy -f

然后在 Windows 系统代理设置中，将代理地址设为 本机IP:3000。浏览器中访问的 HTTP 请求会被转发到目标服务器，HTTPS 请求会建立 CONNECT 隧道。

命令行参数

-t, --target <url>        目标地址（默认 https://api.deepseek.com）
-p, --proxy-port <port>   代理端口（默认 3000）
-w, --web-port <port>     Web UI 端口（默认 8080）
-f, --forward-proxy       启用正向代理（HTTP + CONNECT 隧道）
-u, --upstream <url>      上游代理地址（默认 http://10.30.6.49:9090）
-h, --help                显示帮助信息
-v, --version             显示版本号

Web UI

当前页面即为 Web UI，左侧显示请求列表，点击可查看请求/响应详情。支持 SSE 流式响应、OpenAI/Anthropic 消息格式解析、数据导出/导入等功能。
